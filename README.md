# YouTalk Blog

Учебный проект — блог о психотерапии и ментальном здоровье. Реальный кейс компании YouTalk.

## Что сделано

- Две страницы: `index.html` (список статей) и `article.html` (отдельная статья).
- Семантическая вёрстка (`<header>`, `<main>`, `<footer>`, `<section>`, `<article>`, `<aside>`, `<nav>`).
- БЭМ-методология классов.
- CSS Grid для сетки карточек (3 → 2 → 1 колонки на desktop/tablet/mobile).
- Flexbox для горизонтальных раскладок.
- Адаптивность: 3 брейкпоинта (mobile ≤ 767, tablet 768–1199, desktop ≥ 1200).
- Бургер-меню на мобильных (минимальный ванильный JS).
- Псевдоэлементы (`::after` под пунктами навигации, `::first-letter` для первого абзаца статьи).
- Псевдоклассы (`:hover`, `:focus-visible`, `:active`).
- Accessibility: `aria-*`, `alt` у картинок, контрастный фокус, скрытие декоративных SVG.
- Cursor pointer на всех кликабельных элементах через стандартные стили `<button>` и `<a>`.

## Запуск локально

Файлы статические, никакой сборки не требуется. Достаточно открыть `index.html` в браузере.
Для разработки удобнее запустить локальный сервер (например, через расширение Live Server в VS Code), чтобы корректно работали относительные пути к CSS/JS/иконкам.

```bash
# Любой http-сервер, например:
python3 -m http.server 8000
# затем открыть http://localhost:8000
```

## Что нужно заменить из Figma

Я подготовил каркас, в котором используются плейсхолдеры. Чтобы привести к точному соответствию макету, выполните три шага.

### 1. Картинки

Файлы плейсхолдеров:
- `images/placeholder-hero.png` — иллюстрация в hero на главной.
- `images/placeholder-card.png` — обложки карточек статей и обложка статьи.
- `images/placeholder-thumb.png` — миниатюры в `aside` на странице article.

**Что сделать:**
1. Откройте макет в Figma.
2. Экспортируйте картинки в формате **PNG** (по требованию ТЗ).
3. Сохраните их в папку `images/` с понятными именами (например, `hero.png`, `article-1.png`, `article-2.png`).
4. В `index.html` и `article.html` замените `src="images/placeholder-*.png"` на нужные файлы. Также обновите `alt` на осмысленный.

### 2. SVG-иконки

Все иконки лежат в `icons/`. Если в Figma они нарисованы иначе (другой стиль логотипа, формы бургера и т.п.) — экспортируйте их в **SVG** (по требованию ТЗ) и замените соответствующие файлы. Для inline-SVG в HTML — замените содержимое тега `<svg>`.

### 3. Цвета и шрифты

Все дизайн-токены централизованы в `css/variables.css`. Подгоните значения под Figma:

```css
--color-primary: #2a8e9c;     /* ваш основной цвет */
--color-text: #1c2e3a;        /* ваш цвет текста */
--font-base: "Manrope", ...;  /* ваш шрифт */
```

Если в Figma другой шрифт — поменяйте подключение `<link>` Google Fonts в `<head>` обеих страниц и значение `--font-base`.

## БЭМ-конвенция

- Блок: `header`, `card`, `footer`, `aside`.
- Элемент: `__`, например `header__logo`, `card__title`.
- Модификатор: `_`, например `button_primary`, `header__nav-link_active`.

Двойного дефиса (`--`) не использую — это другая нотация (Two Dashes).

## Адаптивность

| Брейкпоинт | Ширина        | Карточки  | Меню         |
|------------|---------------|-----------|--------------|
| mobile     | до 767px      | 1 колонка | бургер       |
| tablet     | 768–1199px    | 2 колонки | бургер*      |
| desktop    | 1200px+       | 3 колонки | плоское      |

*На tablet можно изменить — снимите `display: none` для `.header__nav` в медиа-запросе `max-width: 1199px`, если хотите показывать плоское меню и на планшетах.

## Accessibility

- Все `<img>` имеют `alt` (пустой `alt=""` для декоративных, осмысленный для содержательных).
- Иконки внутри `<svg>` помечены `aria-hidden="true"`, когда они декоративные.
- Кликабельные иконки в кнопках обёрнуты в `<button aria-label="...">` или ссылку с `aria-label`.
- Бургер использует `aria-expanded` и `aria-controls`.
- Хлебные крошки — `<nav aria-label="Хлебные крошки">`.
- Фокусные стили видны (`:focus-visible` с outline).
- Forms имеют `<label>` (видимый или `.visually-hidden`).

## Известные ограничения

- Бургер-меню требует JS (`js/menu.js`, ~15 строк) — без него меню на мобильных не открывается. Можно переделать на чисто CSS-решение через `:has()` или `<input type="checkbox">`, но JS-вариант проще и доступнее.
- Шапка и подвал дублируются между `index.html` и `article.html` — без сборщика (Vite/Eleventy) это норма для статических HTML. Если потребуется развитие — стоит подключить генератор.
- Плейсхолдеры умышленно SVG, чтобы было видно, что нужно заменить. Финальные картинки — PNG из Figma.
